<objective>
Externalize all hardcoded secrets from config.yaml to environment variables and add startup validation.

Purpose: Prepare the application for cloud deployment by ensuring no secrets are committed to version control. This is the foundation for all subsequent deployment phases (PostgreSQL migration, hosting).

Output: Clean config.yaml (no secrets), comprehensive .env.example, validated .env with all secrets, startup validation in config.py.

Note on ENV-01 scope: ENV-01 lists DATABASE_URL among eventual target secrets, but the current app uses SQLite via a file path (database.path in config.yaml), not a connection string. DATABASE_URL will be introduced in Phase 11 (PostgreSQL migration). CSRF secret is derived from SECRET_KEY via config.py, so validating SECRET_KEY covers CSRF. This plan covers the 4 secrets that exist today: SECRET_KEY, TELEGRAM_BOT_TOKEN, BREVO_SMTP_USERNAME, BREVO_SMTP_PASSWORD.
</objective>

  <action>
    Add a `_validate_required_secrets` method to the `Config` class that checks required secrets are non-empty after all override logic runs. Call it at the end of `Config.load()` before `return config`.

    The method should:
    1. Check that `config.telegram.bot_token` is non-empty (required for Telegram bot functionality)
    2. Check that `config.app.secret_key` is non-empty (required for token signing -- unsubscribe, CSRF, verification, password reset)
    3. Check that `config.email.password` is non-empty (required for SMTP email delivery)
    4. If ANY required secret is missing, raise `ValueError` with a clear message listing ALL missing secrets and pointing to `.env.example`:
       ```
       ValueError: Missing required secrets. Set these environment variables (see .env.example):
         - TELEGRAM_BOT_TOKEN (telegram bot token)
         - SECRET_KEY (application secret key for token signing)
         - BREVO_SMTP_PASSWORD or EMAIL_SMTP_PASSWORD (SMTP password for email delivery)
       ```
    5. Only list the ones that are actually missing (not all three if only one is missing).

    Implementation approach:
    - Add the method as `def _validate_required_secrets(self) -> None:` on the Config class
    - Build a list of (env_var_name, description, config_value) tuples
    - Filter to empty/missing values
    - Raise ValueError if any are missing
    - Call `config._validate_required_secrets()` at end of `Config.load()` just before `return config`

    IMPORTANT: Do NOT validate email.username separately -- it is externalized for consistency (not a secret, just an identifier), not for security. Brevo requires it but other SMTP providers may not. Only validate the three actual secrets listed above.
