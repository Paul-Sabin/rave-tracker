    status_code INTEGER,
    error_message TEXT,
    error_type TEXT,

**fetcher.py modifications:**

Add imports:
- `from ..api.circuit_breaker import circuit_breaker`
- `from ..api.ra_client import RAClient, RAEvent, IPBlockedException`

Modify `Fetcher.fetch_for_rule(self, rule)`:
- Wrap the existing try/except to catch `IPBlockedException` specifically.
- On IPBlockedException: log the error to database via `self.db.log_scraper_error(403, str(e), 'HTTP', circuit_breaker.state, rule.target_name)`. Then re-raise the exception (so fetch_all_rules can handle it).
- On other exceptions: log to database via `self.db.log_scraper_error(None, str(e), 'EXCEPTION', circuit_breaker.state, rule.target_name)`. Return empty list (existing behavior).

Modify `Fetcher.fetch_all_rules(self)`:
- At the start: check `circuit_breaker.should_allow_fetch()`. If False, log warning "Fetch blocked by circuit breaker" and return empty dict.
- Wrap the loop over rules in a try/except for IPBlockedException:
  - On IPBlockedException: log "IP blocked during fetch cycle, aborting", set `fetch_succeeded = False`, break out of loop.
- After the loop: if all rules fetched successfully (no exceptions), call `circuit_breaker.record_success()`. Otherwise call `circuit_breaker.record_failure(error_info)` where error_info is a dict with the last error details.
- Track `fetch_succeeded` boolean, starting True, set to False on any exception during the loop.

Add module-level function `run_fetch()` update:
- The existing `run_fetch()` function creates a new Fetcher and calls fetch_all_rules. This is fine -- circuit breaker integration is inside Fetcher now.
