1. **Add /health endpoint to create_app()**: Inside the create_app() function, AFTER the router includes and BEFORE the return statement, add:

   ```python
   @app.get("/health")
   def health_check(response: Response):
       """Health check endpoint for load balancers and monitoring.

       Returns 200 if healthy, 503 if database unavailable.
       Not protected by auth or CSRF.
       """
       from ..database import get_db

       health_status = {
           "status": "healthy",
           "database": "unknown",
       }

       try:
           db = get_db()
           with db.get_connection() as conn:
               cursor = conn.execute("SELECT 1")
               result = cursor.fetchone()
               if result:
                   health_status["database"] = "connected"
               else:
                   health_status["database"] = "error"
                   health_status["status"] = "unhealthy"
                   response.status_code = 503
       except Exception as e:
           health_status["status"] = "unhealthy"
           health_status["database"] = "disconnected"
           health_status["error"] = str(e)
           response.status_code = 503

       return health_status
   ```

2. **CSRF exemption**: The /health endpoint uses GET, not POST, so CSRF middleware should not interfere. Verify CSRFMiddleware only checks POST/PUT/DELETE/PATCH methods. Looking at existing CSRF implementation: it checks `if request.method in ("POST", "PUT", "DELETE", "PATCH")`. GET is exempt - no change needed.

3. **Auth exemption**: The /health endpoint does not use `require_auth` dependency, so it is public by default. No change needed.

4. **Response format**: Returns JSON:
   - Success: `{"status": "healthy", "database": "connected"}` with 200
   - Failure: `{"status": "unhealthy", "database": "disconnected", "error": "..."}` with 503

5. **Ensure load_dotenv is called**: At the top of create_app(), add `from dotenv import load_dotenv; load_dotenv()` to ensure environment variables are available when gunicorn imports the app directly (bypassing main.py). This is idempotent and safe.
