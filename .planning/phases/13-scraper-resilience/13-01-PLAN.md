  <action>
Create new module `ra-tracker/ra_tracker/api/circuit_breaker.py` with:

1. **CircuitBreakerStatus dataclass** (for admin dashboard consumption):
   - `state: str` (CLOSED, OPEN, HALF_OPEN)
   - `failure_count: int`
   - `last_success: Optional[datetime]`
   - `last_failure: Optional[datetime]`
   - `cooldown_duration: int` (seconds)
   - `cooldown_remaining: Optional[int]` (seconds, None if not OPEN)
   - `error_count_since_success: int`

2. **CircuitBreaker class**:
   - `__init__(self)`: Initialize `state = "CLOSED"`, `failure_count = 0`, `last_success = None`, `last_failure = None`, `cooldown_duration = 3600` (1 hour), `max_cooldown = 86400` (24 hours), `error_count_since_success = 0`, `recent_errors = []` (list for last 10 errors).

   - `should_allow_fetch(self) -> bool`:
     - CLOSED: return True.
     - OPEN: Check if `(datetime.now() - last_failure).total_seconds() >= cooldown_duration`. If yes, transition to HALF_OPEN, log it, return True. If no, log remaining cooldown, return False.
     - HALF_OPEN: return True (allow single probe).

   - `record_success(self)`:
     - Set state = CLOSED, failure_count = 0, error_count_since_success = 0, last_success = datetime.now(), cooldown_duration = 3600 (reset to 1h).
     - Log "Fetch cycle successful. Circuit breaker CLOSED."

   - `record_failure(self, error_info: Optional[dict] = None)`:
     - Increment failure_count and error_count_since_success.
     - Set last_failure = datetime.now().
     - If error_info provided, append to recent_errors (keep last 10 via deque or list slicing).
     - If state == HALF_OPEN: transition to OPEN, double cooldown (min of cooldown*2 and max_cooldown). Log "Probe failed, doubling cooldown."
     - Elif failure_count >= 3: transition to OPEN. Log "Circuit breaker tripped after N failures."
     - Else: stay CLOSED. Log "Fetch cycle failed (N/3)."

   - `_double_cooldown(self)`: `self.cooldown_duration = min(self.cooldown_duration * 2, self.max_cooldown)`.

   - `get_status(self) -> CircuitBreakerStatus`:
     - Compute cooldown_remaining if OPEN: `max(0, int(cooldown_duration - elapsed))`.
     - Return populated CircuitBreakerStatus.

   - `force_close(self)`:
     - Set state = CLOSED, failure_count = 0. Log "Circuit breaker FORCE CLOSED by admin."
     - Do NOT reset error_count_since_success or recent_errors (preserve for diagnostics).

   - `get_recent_errors(self) -> list`: Return copy of recent_errors list.

3. **Module-level singleton**: `circuit_breaker = CircuitBreaker()` -- global instance, in-memory, resets on app restart (per user decision).

Use standard library only (datetime, dataclasses, logging, typing). No external dependencies.
