2. **GET /welcome/step/{step}** — Render step stub page.
```python
@router.get("/welcome/step/{step}", response_class=HTMLResponse)
async def welcome_step(
    request: Request,
    step: int,
    user: User = Depends(require_verified_email),
):
    """Render wizard step page. Step is clamped to [1, 4]."""
    templates = get_templates(request)
    step = max(1, min(step, 4))
    return templates.TemplateResponse(
        "welcome.html",
        {
            "request": request,
            "user": user,
            "csrf_token": getattr(request.state, 'csrf_token', ''),
            "step": step,
        },
    )
```
Key: Clamp step with `max(1, min(step, 4))` — success criterion 3 requires /welcome/step/99 to return step 4, not 404.

3. **POST /welcome/complete** — Mark onboarding done, redirect to dashboard.
```python
@router.post("/welcome/complete")
async def welcome_complete(
    request: Request,
    user: User = Depends(require_verified_email),
):
    """Mark onboarding complete and redirect to dashboard."""
    db = get_db()
    db.set_onboarding_completed(user.id)
    return RedirectResponse(url="/", status_code=303)
```
Use `get_db()` (not `request.app.state.db`) — consistent with all other routes. Use 303 for POST→GET redirect.

**All three routes use `Depends(require_verified_email)`** which handles:
- Unauthenticated → redirect to /login (success criterion 4)
- Unverified email → redirect to /verify-email (success criterion 5)

No imports needed — all required imports (`RedirectResponse`, `HTMLResponse`, `Depends`, `require_verified_email`, `get_db`, `User`, `get_templates`) are already in routes.py.

**Create `ra-tracker/ra_tracker/web/templates/welcome.html`:**

Single stub template that branches on `step` variable. Per CONTEXT.md locked decisions:
- Shows "Step N of 4" only — no content hints or form placeholders
- Step 1-3: "Next" link navigating to next step
- Step 4: "Complete" button that POSTs to /welcome/complete
- No Back button, no progress indicator

```html
{% extends "base.html" %}

{% block title %}Welcome - Rave Tracker{% endblock %}

{% block content %}
<div class="max-w-md mx-auto py-8">
  <div class="card text-center">
    <p class="text-text-muted mb-6">Step {{ step }} of 4</p>
    {% if step < 4 %}
      <a href="/welcome/step/{{ step + 1 }}" class="btn btn-primary">Next</a>
    {% else %}
      <form action="/welcome/complete" method="POST">
        <input type="hidden" name="csrf_token" value="{{ csrf_token }}">
        <button type="submit" class="btn btn-primary">Complete</button>
      </form>
    {% endif %}
  </div>
</div>
{% endblock %}
```

The CSRF hidden field is required — without it POST /welcome/complete returns 403 (CSRFMiddleware validates form field).

**Anti-patterns to avoid:**
- Do NOT add an `onboarding_completed` check to the route — CONTEXT.md says "Allow re-entry: users who already completed onboarding can revisit /welcome"
- Do NOT use 301 permanent redirect for GET /welcome
- Do NOT create separate template files per step
- Do NOT put routes in a new file — all web routes live in routes.py
