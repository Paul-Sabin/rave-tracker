    **1. Add scraper_fetch_log table to database.py:**

    Add to BOTH `SCHEMA` (SQLite) and `PG_SCHEMA` (PostgreSQL) strings, after the existing `scraper_health_log` table:

    SQLite version:
    ```sql
    CREATE TABLE IF NOT EXISTS scraper_fetch_log (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        started_at DATETIME NOT NULL,
        completed_at DATETIME,
        duration_seconds REAL,
        events_found INTEGER DEFAULT 0,
        rules_processed INTEGER DEFAULT 0,
        status TEXT NOT NULL DEFAULT 'RUNNING',
        error_message TEXT,
        circuit_breaker_state TEXT
    );
    CREATE INDEX IF NOT EXISTS idx_scraper_fetch_started ON scraper_fetch_log(started_at DESC);
    ```

    PostgreSQL version:
    ```sql
    CREATE TABLE IF NOT EXISTS scraper_fetch_log (
        id SERIAL PRIMARY KEY,
        started_at TIMESTAMP NOT NULL,
        completed_at TIMESTAMP,
        duration_seconds REAL,
        events_found INTEGER DEFAULT 0,
        rules_processed INTEGER DEFAULT 0,
        status TEXT NOT NULL DEFAULT 'RUNNING',
        error_message TEXT,
        circuit_breaker_state TEXT
    );
    CREATE INDEX IF NOT EXISTS idx_scraper_fetch_started ON scraper_fetch_log(started_at DESC);
    ```

    **2. Add database methods to Database class:**

    `start_scraper_fetch(self) -> int`:
    - Insert row with started_at=CURRENT_TIMESTAMP (use `datetime.utcnow()` for consistency), status='RUNNING'
    - Return the inserted row ID
    - Use `self.ph` for parameter placeholders (? for SQLite, %s for PostgreSQL)
    - For returning the ID: SQLite uses `cursor.lastrowid`, PostgreSQL needs `RETURNING id`

    `complete_scraper_fetch(self, fetch_id: int, events_found: int, rules_processed: int, status: str, error_message: str = None)`:
    - Calculate duration from started_at to now
    - Update row: set completed_at, duration_seconds, events_found, rules_processed, status, error_message, circuit_breaker_state
    - Get circuit_breaker_state from `circuit_breaker.state` import
    - Duration: query started_at first, compute `(datetime.utcnow() - started_at).total_seconds()`

    `get_scraper_health_summary(self, days: int = 7) -> dict`:
    - Query scraper_fetch_log for aggregate stats over last N days
    - Return: total_fetches, successful (status='SUCCESS'), failed (status='FAILURE'), avg_duration_seconds, total_events_found, last_successful_fetch (max started_at WHERE status='SUCCESS')
    - Use appropriate date arithmetic for SQLite vs PostgreSQL:
      - SQLite: `started_at > datetime('now', '-{days} days')`
      - PostgreSQL: `started_at > NOW() - INTERVAL '{days} days'`
    - Handle the dual-mode pattern using `self.is_postgres` check

    `get_recent_fetch_history(self, limit: int = 20) -> list`:
    - Query scraper_fetch_log ordered by started_at DESC, limit N
    - Return list of dicts with all columns

    `get_fetch_success_rate_trend(self, days: int = 7) -> list`:
    - Daily success rate over last N days
    - Group by date, count total and successful, calculate percentage
    - Return list of dicts: [{date, total, successful, success_rate}]
    - For date grouping: SQLite uses `DATE(started_at)`, PostgreSQL uses `DATE(started_at)` (same)

    `cleanup_old_fetch_logs(self, days: int = 30)`:
    - Delete scraper_fetch_log entries older than N days
    - Similar pattern to existing `cleanup_old_scraper_logs`
