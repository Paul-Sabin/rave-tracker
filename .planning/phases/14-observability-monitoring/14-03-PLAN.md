    Create `ScraperAlerter` class:

    `__init__(self)`:

    - Just validates config availability

    `check_and_alert(self, fetch_status: str)`:
    - Main entry point, called after each fetch cycle
    - Get config (need telegram.bot_token and telegram.chat_id for admin)
    - Get db
    - If bot_token or chat_id not configured, return early (no alerting possible)

    Logic:
    ```python
    def check_and_alert(self, fetch_status: str):
        """Check scraper state and send alerts if needed.

        Args:
            fetch_status: 'SUCCESS', 'FAILURE', or 'SKIPPED'
        """
        config = get_config()
        db = get_db()

        # Need bot token and admin chat_id for alerts
        if not config.telegram.bot_token or not config.telegram.chat_id:
            return

        alert_state = db.get_scraper_alert_state()

        if fetch_status == 'SUCCESS':
            # Reset consecutive failures
            if alert_state['consecutive_failures'] > 0:
                failures_recovered_from = alert_state['consecutive_failures']
                db.reset_consecutive_failures()

                # Send recovery notification if alert was sent
                if alert_state['alert_sent']:
                    self._send_recovery(config, failures_recovered_from)
                    db.set_scraper_alert_sent(False)

        elif fetch_status in ('FAILURE', 'SKIPPED'):
            # Increment consecutive failures
            new_count = alert_state['consecutive_failures'] + 1
            db.update_consecutive_failures(new_count)

            # Alert on threshold (3+ failures) if not already sent
            if new_count >= FAILURE_THRESHOLD and not alert_state['alert_sent']:
                self._send_failure_alert(config, new_count)
                db.set_scraper_alert_sent(True, message=f"Alert sent after {new_count} consecutive failures")
    ```

    `_send_failure_alert(self, config, failure_count: int)`:
    - Create Bot instance with config.telegram.bot_token
    - Send message to config.telegram.chat_id (admin chat)
    - Message format per user decision context:
      ```
      Scraper Failure Alert

      {failure_count} consecutive fetch failures detected.
      Circuit breaker state: {circuit_breaker.state}

      Check /admin/scraper-status for details.
      ```
    - Use `asyncio` to run the async send: create event loop or use existing
    - Wrap in try/except, log errors but don't raise

    `_send_recovery(self, config, failures_recovered_from: int)`:
    - Send message to admin:
      ```
      Scraper Recovered

      Scraper recovered after {failures_recovered_from} failures.
      System operating normally.
      ```
    - Same async pattern as failure alert

    For the async Telegram send, use the pattern from existing `notifier.py` `_run_async()`:
    ```python
    def _send_telegram(self, bot_token: str, chat_id: str, message: str):
        """Send Telegram message synchronously (called from scheduler thread)."""
        async def _send():
            bot = Bot(token=bot_token)
            await bot.send_message(
                chat_id=chat_id,
                text=message,
                disable_notification=False,  # Admin alerts should notify
            )

        try:
            loop = asyncio.new_event_loop()
            asyncio.set_event_loop(loop)
            loop.run_until_complete(_send())
            loop.close()
        except Exception as e:
            logger.error(f"Failed to send Telegram alert: {e}")
    ```

    Create module-level singleton:
    ```python
    scraper_alerter = ScraperAlerter()
    ```

    **NOTE:** Per user decision, alert fires via Telegram ONLY (no email). Alert fires once and silences until recovery. Recovery notification sent when scraper starts working again.
