<task type="checkpoint:human-action" gate="blocking">
  <name>Task 1: Configure custom domain DNS records</name>
  <files></files>
  <action>
    The user must configure DNS records at their domain registrar to point to the Render service. This cannot be automated because it requires access to the domain registrar's dashboard.

    Steps for the user:

    1. Add custom domain in Render:
       - Go to Render Dashboard -> rave-tracker-web -> Settings -> Custom Domains
       - Click "Add Custom Domain"
       - Enter your domain (e.g., ravetracker.whotrustswho.com or ravetracker.example.com)
       - Render will display the required DNS records

    2. Configure DNS at your registrar:
       For a subdomain (e.g., ravetracker.example.com):
       - Add a CNAME record:
         Name: ravetracker (or your chosen subdomain)
         Value: The .onrender.com hostname shown by Render (e.g., rave-tracker-web.onrender.com)
         TTL: 300 (or lowest available)
       For an apex domain (e.g., example.com):
       - If your registrar supports ALIAS/ANAME records: Add an ALIAS record pointing to the Render hostname
       - If not: Use a CNAME flattening service or add a subdomain instead

    3. Wait for DNS propagation:
       - DNS changes can take 5 minutes to 48 hours to propagate
       - Check propagation with: nslookup ravetracker.example.com or https://dnschecker.org
       - Render will automatically issue a Let's Encrypt SSL certificate once DNS is verified

    4. Verify SSL certificate:
       - In Render Dashboard -> Custom Domains, wait for status to change from "Certificate pending" to "Active"
       - This typically takes 5-30 minutes after DNS propagation

    5. Update BASE_URL environment variable:
       - Go to Render Dashboard -> rave-tracker-web -> Environment
       - Update BASE_URL to your custom domain (e.g., https://ravetracker.whotrustswho.com)
       - This ensures email links (verification, password reset, unsubscribe) point to the correct domain
       - The service will auto-redeploy with the new value
  </action>
  <verify>User confirms SSL certificate is "Active" in Render dashboard and the custom domain loads the application</verify>
  <done>Custom domain configured with DNS records pointing to Render. SSL certificate issued by Let's Encrypt. BASE_URL updated to custom domain.</done>
</task>
