def get_all_rules(self, user_id: Optional[int] = None) -> List[Rule]:
    """Get all rules, optionally filtered by user."""
    with self.get_connection() as conn:
        if user_id is not None:
            cursor = conn.execute(
                "SELECT * FROM rules WHERE user_id = ? ORDER BY rule_type, target_name",
                (user_id,)
            )
        else:
            cursor = conn.execute("SELECT * FROM rules ORDER BY rule_type, target_name")
        return [self._row_to_rule(row) for row in cursor.fetchall()]

  <name>Task 2: Add user-scoped event and stats methods</name>

  <action>
Add methods for user-scoped event retrieval and statistics:

1. **get_upcoming_events_for_user(user_id: int) -> List[Event]:**
   - Get events that match the user's rules (via event_rules JOIN)
   - Only include events with date >= today
   - Load matched_rules for each event but ONLY rules belonging to this user
   - Query pattern:
   ```sql
   SELECT DISTINCT e.* FROM events e
   INNER JOIN event_rules er ON e.id = er.event_id
   INNER JOIN rules r ON er.rule_id = r.id
   WHERE r.user_id = ? AND e.date >= ?
   ORDER BY e.date, e.start_time
   ```
   - For each event, load artists and promoters (shared data)
   - For matched_rules, only include this user's rules:
   ```sql
   SELECT r.* FROM rules r
   JOIN event_rules er ON r.id = er.rule_id
   WHERE er.event_id = ? AND r.user_id = ?
   ```

2. **get_user_stats(user_id: int) -> dict:**
   - Return user-scoped statistics:
     - active_rules: COUNT of user's active rules
     - upcoming_events: COUNT of events matching user's rules
     - notifications_sent: COUNT of notifications for user's rules
   - Query patterns:
   ```sql
   -- Active rules for user
   SELECT COUNT(*) FROM rules WHERE is_active = 1 AND user_id = ?

   -- Upcoming events matching user's rules
   SELECT COUNT(DISTINCT e.id) FROM events e
   INNER JOIN event_rules er ON e.id = er.event_id
   INNER JOIN rules r ON er.rule_id = r.id
   WHERE r.user_id = ? AND e.date >= ?

   -- Notifications for user's rules
   SELECT COUNT(*) FROM notifications WHERE user_id = ?
   ```

3. **count_legacy_data(user_id: int) -> dict:**
   - Count rules and notifications that were migrated to this user
   - Legacy data = records created before user's created_at timestamp
   - Returns: {"rules": int, "notifications": int}
   - Used for dashboard welcome message
   - Query pattern:
   ```sql
   SELECT COUNT(*) FROM rules
   WHERE user_id = ? AND created_at < (SELECT created_at FROM users WHERE id = ?)
   ```

4. **Update add_notification to include user_id:**
   - Modify add_notification(event_id, rule_id, user_id: Optional[int] = None)
   - Store user_id when creating notification record
   - Allows per-user notification filtering
